# go-api-changelog

Tracks when each Camunda 8 REST API operation and property was first introduced, by parsing OpenAPI specs across versions 8.5–8.9.

Produces `output/version-map.json` — a map of every operation and request/response property to the version it first appeared.

## Usage

```bash
npm install
bash extract-specs.sh   # copies bundled specs from camunda-go-client
node build-version-map.mjs
```

Or all at once:

```bash
npm run all
```

## Prerequisites

- Bundled OpenAPI specs in `../camunda-go-client/spec/{8.5,8.6,8.7,8.8,8.9}/bundled-api.yaml`
  (generated by the [camunda-go-client](../camunda-go-client) project)
- Node.js 18+

## Output Format

`output/version-map.json`:

```json
{
  "operations": {
    "GET /topology": { "version": "8.5", "summary": "Get cluster topology." },
    "POST /jobs/activation": { "version": "8.6", "summary": "Activate jobs" }
  },
  "properties": {
    "POST /process-instances > request > variables": {
      "version": "8.6", "type": "object", "location": "request",
      "endpoint": "POST /process-instances", "property": "variables", "depth": 0
    }
  },
  "deletedOperations": {
    "POST /document/{documentId}/links": {
      "removedIn": "8.7", "summary": "Create document link (alpha)"
    }
  }
}
```

## How It Works

1. **extract-specs.sh** — Copies the bundled OpenAPI YAML specs from `camunda-go-client/spec/` for each version.
2. **build-version-map.mjs** — Parses each spec, extracts operations (method + path) and their request/response properties (resolving `$ref` and `allOf`), then walks versions in order to record when each first appeared.

## Comparison with api-added-in-analysis

| Aspect | go-api-changelog | api-added-in-analysis |
|--------|-----------------|----------------------|
| Approach | Direct spec parsing | openapi-diff between version pairs |
| Operations | 185 (identical) | 185 (identical) |
| Properties | 1,283 | 269 |
| Version accuracy | First appearance in spec | First appearance in diff report |

Operations match perfectly. Properties are more comprehensive here because we scan every spec directly rather than relying on diff reports which only capture changes between consecutive pairs.
